• Abandon Locality: Frame-Wise Embedding Aided Transformer for Automatic Modulation Recognition

    Item Type Journal Article
    Author Yantao Chen
    Author Binhong Dong
    Author Cuiting Liu
    Author Wenhui Xiong
    Author Shaoqian Li
    Abstract Automatic modulation recognition (AMR) has been considered as an efﬁcient technique for non-cooperative communication and intelligent communication. In this work, we propose a modiﬁed transformer-based method for AMR, called frame-wise embedding aided transformer (FEA-T), aiming to extract the global correlation feature of the signal to obtain higher classiﬁcation accuracy as well as lower time cost. To enhance the global modeling capability of the transformer, we design a frame-wise embedding module (FEM) to aggregate more samples into a token in the embedding stage to generate a more efﬁcient token sequence. We also present the optimal frame length by analyzing the representation ability of each transformer layer for a better trade-off between the speed and the performance. Moreover, we design a novel dual-branch gate linear unit (DB-GLU) scheme for the feed-forward network of the transformer to reduce the model size and enhance the performance. Experimental results on RadioML2018.01A datasets demonstrate that the proposed method outperforms state-of-the-art works in terms of recognition accuracy and running speed.

      “In general, the modulated signals usually transmit bit sequences without semantic information, and each modulation symbol appears in the waveform with uniform probability. Hence, more valuable features for the AMR task should be uniformly distributed in the waveform. The extract of these features should not be within the locality bias assumption.” (Chen et al., 2023, p. 327) 1. 통신 신호(Modulated Signals)이 자연어 데이터처럼 뚜렷한 의미론적(Semantic) 국소 정보가 없으며, 각 변조 심볼(Symbol)이 파형 전체에 걸쳐 균일한 확률분포로 존재한다는 '물리적 직괸'

      2. 이에 의거하여, Inductive Bias는 사용할 필요가 없으며, Global Correlation을 추출하는 것이 AMC 성능 향상과 연산 속도 개선에 유리하다는 가설 도출

      3. 따라서 Transformer의 병렬 연산 능력을 극대화하여 추론 속도를비약적으로 높히자는 결론.

      “The contributions of this study are summarized as follows.” (Chen et al., 2023, p. 327) [contribution]
      1. Inductive Bias를 극복하고 Transformer만 사용

      2. 경량화

      3. Inference Time 최소화


      (Chen et al., 2023, p. 328) [GLU(Gated Linear Units)]

      1. 본래 FFN에서 activation function(ReLU, GeLU 등등)을 대체하여, 표현력을 더 높히기 위한 방법.

      2. 비선형선에 대한 표현력이 더 높아지는 대신, 연산량이 늘어남(Trade-off)

      3. d'_ff차원을 2/3으로 줄임으로써 증가한 매개변수 수와 계산량을 원래 수준으로 맞출 수 있음

      4. 수식 = W_fc1{W_fc1*X_l ⊗ g(W_fc2*X_l)}


      (Chen et al., 2023, p. 328) [DB-GLU]
      1. GLU를 개선하여, 더 높은 FFN에서 표현력 확보.

      2. W(linear projection)의 차원 d_f = 4L(MLP의 절반 수준: MLP는 8L)으로 설정하여, MLP에 비해 (1/2)^2=(1/4)=25%수준의 parameters만 사용

      3. 이러한 parameters 다이어트를 통해, 낮은 수준의 FLOPs과 parameters 유지.


      (Chen et al., 2023, p. 328) [전체 구조]
      1. CNN 앞단을 제거함으로써, 파라미터 수를 획기적으로 낮춤(=Abandon locality, 대신 저SNR에서 급격한 성능 하락)

      2. FEM을 통해 Transformer로 입력되는 token sequence 수를 획기적으로 제한함으로싸, 연산 복잡도를 낮춤

      3. Transformer Encoder block에서 FFN을 MLP 대신 DB-GLU를 설계하고 사용함으로써, 표현력을 극대화하고 파라미터 수를 획기적으로 낮춤

      4. 해당 3가지의 주요구조로, 순수한 MHSA를 사용함에도 불구하고, 매우 낮은 파라미터 수와 FLOPs, Inference time을 기록함.


      (Chen et al., 2023, p. 328) [FEM module]
      1. input signal r을 받아 x sequence만듦

      2. Framing을 통해 X tensor로 변환

      3. Linear matrix를 통과시키고, class token concatenation

      4. Absolute Positional Embedding 기법 활용

      5, 최종 input token X_0 [F+1, 2L]: F+1(프레임 길이 + 클래스 토큰), 2L(I/Q 각 길이는 L, 두 신호 concatenate)

      6. 해당 FFM을 통해 Transformer Encoder block으로 입력되는 token sequence 자체를 매우 낮은 수준으로 유지했기에, MHSA에서 낮은 FLOPs를 기록함.


      (Chen et al., 2023, p. 328) [Received signal 수식]
      1. s(t): transmission symbol(송신 신호)

      2. m: modulation 종류

      3. F: modulator

      4. h(t): impulse response

      5. n: noise

      6. AMC: s를 통해 m을 유추하는 것


      (Chen et al., 2023, p. 328) [MHSA block]
      1. head 수 h만큼 MHSA 적용

      2. talking heads 기법 적용

      3. 각 head를 concatenate 시키고 output linear projection W_o에 적용

      4. 최종 output size [F+1, 2L]

      “input signal r ∈ CN×1” (Chen et al., 2023, p. 328) 1. N: length of signal

      “Let L and R denote the frame size and the sliding step length” (Chen et al., 2023, p. 328) 1. L: frame size

      2. R: sliding step length

      “x(t) = [rI (tR), rI (tR + 1), . . . , rI (tR + L − 1), rQ(tR), rQ(tR + 1), . . . , rQ(tR + L − 1)]T” (Chen et al., 2023, p. 328) [개별 프레임 column vector]
      1. tR: 시작점, t(현재 프레임의 순번: 0번째, 1번째 ...), R(보폭), tR(해당 t번째 프레임이 전체 시퀀스 길이의 어디서부터 시작하는지 나타냄)

      2. +1씩: 시퀀스를 한 칸씩 이동 시킴

      3. L-1: 끝점(총 길이=L)

      4. 차원: 2L(I와 Q를 concatenate) x 1(column vector)

      “X = [x(0), x(1), . . . , x(F −1)]T ,” (Chen et al., 2023, p. 328) [Input feature]
      1. 개별 column vector x를 concatenate

      2. 차원: F(전체 프레임 길이) x 2L(개별 프레임 column vector 크기)

      “the learnable position bias Wpos ∈ R(F +1)×2L for position embedding” (Chen et al., 2023, p. 328) [Positional Embedding]
      1.Learnable position bias Matrix를 사용한 Absolute positional embedding 방식

      “X0 = [xcls; XWe] + Wpos.” (Chen et al., 2023, p. 328) [최종 input token sequence]

      “For the MHSA block, we incorporate the talking-head attention [15] as the implementation scheme.” (Chen et al., 2023, p. 328) [MHSA Block]
      1. O(N^2) 연산 복잡도 Attention 기법 활용

      2. talking-head 기법 활용: head가 뽑아내는 특징 간 융합 높힘

      “To compress the model size for the lightweight requirement of the AMR task, we propose a practical DB-GLU scheme to replace the traditional MLP scheme in the FFN” (Chen et al., 2023, p. 328) [Feed Forward Network]
      1. 기존의 MLP 방식 대신 연산량을 낮추기 위한 DB-GLU기법 활용


      (Chen et al., 2023, p. 329) 1. P_i(Dot product) = Q*K/(d_p)^1/2, [F+1, F+1]

      2. P_tilda_i(context attention matrix): W_t(head-wise linear mixing)을 통해 talking heads구현

      3. X_i_l: i번째 헤드에 대한 feature vector, X_l-1(이전 layer에서 넘어오는 입력 텐서), W^i_v(i번째 헤드의 value matrix), softmax(p_i)를 통해 각 토큰이 서로의 정보를 참조하는 확률값으로 변환.
      i번째 헤드의 feature vector가 Attention 비율대로 V값을 가져감.
      사이즈: [F+1, d_p]

      “Finally, all of the heads X(i) l ∈ R(F +1)×dp are concatenated in the second dimension, and a linear projection Wo ∈ Rhdp×2L is applied.” (Chen et al., 2023, p. 329) 1. 각 헤드의 feature maps을 concatenate 시키고 [F+1, h*d_p]

      2. 이를 linear projection W_o에 적용

      3. 최종 output 사이즈: [F+1, 2L]


      (Chen et al., 2023, p. 329) [DB-GLU 수식]
      1. 표현력을 극대화 하기 위해, 새로운 GLU 구조 설계

      2. W(linear projection)의 차원 d_f = 2L(MLP의 절반 수준: MLP는 원래 4L)으로 설정하여, MLP에 비해 (1/2)^2=(1/4)=25%수준의 parameters만 사용

      “36L2 + 16L + h2 = O(L2)” (Chen et al., 2023, p. 329) [Computational Complexity]
      1. L(frame length)의 제곱에 비례

      “O(LF 2)” (Chen et al., 2023, p. 329) [Computational Complexity]
      1. MHSA은 F(#frame)의 제곱에 비례

      2. L과 F는 반비례하므로, 최적점을 찾아야함.


      (Chen et al., 2023, p. 329) 1. W_Q: 각 head에 대한 linear projection[2L,d_p(=2L/h)]를 concatenation[2L,2L]

      2. W_K도 마찬가지


      (Chen et al., 2023, p. 329) [Context Attention scores]
      1. talking heads 기법을 도입: 기존 MHSA에서 P^tilda_I는 d_p의 차원에 같힘(=Low-Rank bottleneck)

      2. talking heads에서 섞임 가중치 W_t가 이상적으로 학습되었다고 가정하면, 특정 헤드의 유사도 행렬 P^tilda_i는 W_Q와W_K의 내적을 한 것과 동일함.

      3. 따라서 결과적으로 W_Q와 W_K의 Rank인 2L로 커짐

      4. 이제 X_l-1[F+1,2L]의 Rank와 결합하여, Rank(P^tilda_i) = min(Rank(X_l-1),Rank(W_Q*W_K))=min(F+1, 2L)로 결정됨.

      5. Transformer의 token sequence 수 = F+1이므로, F+1<=2L을 만족해야, Low-Rank bottleneck을 피할 수 있음.

      6. 해당 관계식을 통해 L_opt 도출 가능


      (Chen et al., 2023, p. 329) [L_opt]
      1. Row-rank bottleneck을 피하기 위해, F+1<=2L을 만족해야함.

      2. 이를 F = (N-L)/R+1의 수식에 대입하면, L_opt를 구할 수 있음.

    • 용어

      • Rank(선형대수학):

        행렬의 column이 만들어내는 벡터 공간에서 basis의 최대 개수이자, 그 공간의 Dimension(서로 독립적인 차원을 가지는 basis의 수).

      • Low-Rank Bottleneck:

        S(시퀀스 길이)는 넓은데, 헤드 차원(d_p)가 너무 적어서 MHSA에서 [S,S]의 다채로운 관계를 표현하지 못하는 현상.

        Rank inequality:

        [rank(AB)<=min(rank(A),rank(B))]관계 성립.

        계산의 효율성을 위해, Transformer에서 일반적으로 d_p를 S보다 작게 설정한다(Q와 K [S,d_p]).

        이를 Rank inequality 수식에 대입하면, P(Context Attention Matrix, [S,S])의 최대 rank가 d_p로 결정된다.

        만약 S = 128, d_p =16이라면, [128, 128] Matrix(128 시퀀스 간의 관계)를 단 16가지 패턴으로 제한시켜 나타내야 한다(=Low-Rank Bottleneck).
